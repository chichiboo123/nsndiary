# 나세나반 다이어리 📒

초등학교 담임교사를 위한 학급 경영 통합 관리 웹앱입니다.  
Google Apps Script + Google Sheets 기반으로 별도 서버 없이 무료로 운영할 수 있습니다.

---

## 주요 기능

### 🏠 홈 & 전체 검색
- 수업 기록, 일상 기록, 학생 기록, 상담 일지, 일정 등 전 모듈 통합 키워드 검색
- 검색 결과 유형 뱃지·미리보기 표시, 클릭 시 해당 페이지로 이동
- 주요 기능 빠른 이동 버튼 그리드

### 📅 학급 및 학교 일정
- 일정 목록·월별 캘린더 두 가지 뷰 제공
- 카테고리별(1~6교시, 업무 등) 색상 뱃지
- 주간학습안내 이미지 업로드·확대 보기
- 실시간 아날로그 시계 표시

### 👦 우리 반 아이들 (학생 정보)
- 버튼 그리드 / 테이블 목록 두 가지 뷰 전환
- 학생 번호·이름·연락처·보호자 연락처·특이사항 관리
- 학생 카드 클릭 시 종합 기록 대시보드 연결

### 📋 출결 기록
- 질병결석·체험학습·미인정결석·질병조퇴·질병지각·미인정지각·미인정조퇴·기타결석 8개 유형
- **기간 연속 입력**: 시작일~종료일 선택 시 주말·공휴일 자동 제외, 수업일 수 실시간 표시
- 공휴일 데이터: 2025~2027년 법정 공휴일 내장
- 날짜·상태·학생명 필터링, 개별 기록 수정·삭제
- **출결 기록 보관함**: 월별 정산을 위한 3월~12월 월별 필터 버튼, 2열 그리드로 압축 표시
- **목록형 / 달력형 두 가지 보기**: 토스 '내 소비' 스타일 월 달력에서 날짜별 출결 현황·월별 합계 확인, 날짜 클릭 시 상세 기록 표시

### 📝 수업 기록
- 날짜·교시·과목·배움 주제·수업 내용·수업 성찰 입력
- 참고 URL 저장
- 이미지·PDF·HWP·DOCX·XLSX·PPTX 파일 첨부 (파일당 최대 20MB, Google Drive 저장)
- 드래그 앤 드롭 업로드, 이미지 갤러리·문서 임베드 뷰어

### 🌱 교실의 소소한 일상 (일상 기록)
- 날짜·핵심 키워드·내용·관련 URL·첨부 파일 기록
- 날짜 범위·키워드 검색 필터

### 🌟 학생 행동 발달 기록
- 학습 태도·생활 태도·교우 관계·기타 카테고리별 관찰 기록
- 날짜·학생명·카테고리 필터링, 수정·삭제

### 💬 상담 일지
- 다중 학생 선택 (태그 방식)
- 대상: 학생 본인 / 보호자 구분
- 방법: 대면·전화·온라인/메시지 구분
- 날짜·방법·학생명 필터링

### 🧮 수행평가
- 구분·교과/활동영역·단원/주제명·성취기준·평가요소·평가영역·평가방법·주안점·평가시기로 평가 계획 등록·수정
- 평가기준은 매우잘함·잘함·보통·노력요함 수준별 도달 내용 입력
- **엑셀 업로드**: 정해진 양식의 `.xlsx` 파일을 올리면 평가 계획이 한 번에 등록 (헤더 자동 인식, 동일 항목 자동 중복 제거)
- **양식 받기**: 업로드용 빈 엑셀 양식을 예시와 함께 내려받기
- 계획 카드별 학생 평가 결과 입력: 매우잘함·잘함·보통·노력요함·기타(주관식) 5단계
- 계획별 입력 진행률 표시 (n/전체 명)
- 업로드·입력 데이터는 연동된 스프레드시트의 `수행평가계획`·`수행평가결과` 시트에 저장되어 어디서든 확인 가능
- 두 시트는 없을 때만 자동 생성 (기존 시트·데이터는 변경하지 않음)

### 💻 자주 쓰는 디지털 도구
- 카테고리별 도구명·링크·설명 목록
- Google Sheets 시트에서 데이터 관리

### 🪑 우리 반 자리배치
- 구역·행 수 자유 설정, 복수 배치도 저장·전환
- 학생 배정·해제, 무작위 섞기, 되돌리기, 좌우 뒤집기
- 서버 저장 + 브라우저 로컬 저장 이중 백업

### ✅ 제출 확인
- 과제별 제출·미제출 현황 관리
- 학생 버튼 클릭으로 제출 상태 토글
- 완료·미완료 카운트 뱃지, 화면 확대·축소

### 🎨 테마 변경
- 블루·그린·옐로·레드/핑크 4가지 색상 테마
- 선택값 브라우저 로컬 저장으로 유지

직접 만들어 두어야 하는 시트(이름이 정확히 일치해야 함):

| 시트명 | 용도 |
|--------|------|
| 학생정보 | 학생 기본 정보 |
| 일정 | 학급·학교 일정 |
| 출결기록 | 출결 현황 |
| 수업기록 | 수업 내용·성찰 |

| 상담기록 | 상담 일지 |

앱이 **자동으로 생성**하는 시트(직접 만들지 않아도 됨):

| 시트명 | 용도 | 컬럼 구성 |
|--------|------|-----------|
| 수행평가계획 | 수행평가 계획 | ID · 구분 · 교과/활동영역 · 단원/주제명 · 성취기준 · 평가요소 · 평가영역 · 평가방법 · 수업·평가 연계 주안점 · 평가시기 · 매우잘함 · 잘함 · 보통 · 노력요함 |
| 수행평가결과 | 학생별 평가 결과 | ID · 계획ID · 번호 · 이름 · 평가결과 · 기타내용 · 수정일시 |

### 설치 및 배포

1. 이 저장소의 `code.gs`를 Apps Script 프로젝트의 `Code.gs`에, `index.html`을 **`Index`** 라는 이름의 HTML 파일로 복사합니다. (`doGet`이 `Index` 파일을 부르므로 이름이 정확해야 합니다.)
2. 두 파일을 모두 **저장(Ctrl+S)** 합니다.
3. GAS 편집기에서 **배포 → 새 배포** 를 선택합니다.
4. 유형: **웹앱**, 액세스: **나만** 또는 조직 설정 후 배포합니다.
5. 배포된 웹앱 URL로 접속하면 초기 설정 화면이 나타납니다.
6. Google Sheets URL을 입력해 연결합니다.

> **코드를 수정한 뒤에는 반드시 재배포해야 반영됩니다.**
> `배포 → 배포 관리 → (기존 배포 선택) → 연필(편집) → 버전: 새 버전 → 배포`
> 를 실행하세요. 저장만 하고 재배포하지 않으면 `/exec` 주소에서는 이전 코드가 계속 실행됩니다.
> (개발용 `/dev` 테스트 주소는 항상 최신 저장 코드를 실행합니다.)

### 수행평가 시트가 안 만들어질 때 (자주 묻는 문제)

`수행평가계획`·`수행평가결과` 시트는 **`doGet`(웹앱 화면 표시)만으로는 생성되지 않습니다.**
실제로 수행평가 데이터를 읽거나 쓸 때 만들어집니다. 다음 중 한 가지를 하세요.

- **방법 A (권장)** — 배포된 웹앱에서 사이드바 **수행평가** 탭에 들어갑니다. 진입 시
  `getAssessmentData()`가 실행되며 두 시트가 자동 생성됩니다.
- **방법 B (편집기에서 즉시 생성)** — Apps Script 편집기 상단의 함수 선택 메뉴에서
  **`setupAssessmentSheets`** 를 고른 뒤 **[실행]** 버튼을 누릅니다. 처음 실행 시
  Google 권한 승인 창이 뜨면 허용하세요. 실행이 끝나면 스프레드시트에 두 시트가
  헤더와 함께 생성됩니다.

> ⚠️ 직전 버전(컬럼 11개)에서 이미 `수행평가계획` 시트가 만들어졌다면, 현재 버전은
> 컬럼이 14개로 늘었습니다. 데이터가 없다면 기존 `수행평가계획`·`수행평가결과` 시트를
> 삭제한 뒤 위 방법으로 다시 생성하면 새 양식으로 만들어집니다.

### 수행평가 엑셀 업로드 양식

수행평가 탭의 **[양식 받기]** 버튼으로 받은 `.xlsx`(또는 동일 헤더의 파일)를
**[엑셀 업로드]** 하면 평가 계획이 일괄 등록됩니다. 헤더(열 제목)는 다음과 같으며,
열 순서가 바뀌거나 띄어쓰기가 달라도 자동 인식합니다.

```
연번 | 구분 | 교과/활동영역 | 단원/주제명 | 성취기준 | 평가요소 | 평가영역 |
평가방법 | 수업·평가 연계 주안점 | 평가시기 | 매우 잘함 | 잘함 | 보통 | 노력 요함
```

- `단원/주제명`, `성취기준`, `평가요소`가 모두 비어 있는 행은 건너뜁니다.
- 이미 등록된 동일 항목(`단원/주제명`+`성취기준`+`평가요소`)은 자동으로 제외되어
  같은 파일을 다시 올려도 중복되지 않습니다.
